API and data documentation

Product data structure

  • id (string): Unique alphanumeric product identifier, generated automatically.
  • name (string): Product name, 3–50 characters.
  • category (string): One of the predefined categories in the category selector.
  • price (number): Decimal value > 0, up to 2 decimal places.
  • stock (number): Integer, non‑negative.

Operations

  • Load inventory: Data is loaded asynchronously from inventory.json with up to 3 retries. On success, data is cached in memory and backed up to localStorage. On failure, an empty inventory is used.
  • Create product: Submitting the form validates inputs, checks for duplicate names within the same category, generates a unique ID, adds the product to the in‑memory cache, updates localStorage, refreshes the table, and recalculates inventory values.
  • Edit product: Clicking “Edit” enables inline editing of price and stock. Saving validates the new values, records an audit trail entry, updates the cache and localStorage, refreshes the table, and recalculates inventory values.
  • Delete product: Clicking “Delete” asks for confirmation, removes the product from the cache and localStorage, updates the table and inventory values, and exposes an undo action for the most recent deletion.
  • Search and filter: Search input and category filter dynamically update the displayed rows, preserving active filters during add, edit, delete, and undo operations.
  • Export: Export buttons generate JSON or CSV files for the current inventory, and a CSV summary of inventory value by category.
